# Исправление проблемы с выходом из системы

## Проблема
При нажатии кнопки "Выйти" пользователя не перенаправляло на экран входа.

## Причина
В файле `more_screen.dart` функция выхода была не реализована (TODO комментарий), и просто делался переход на `/login` без вызова метода `logout()` из `authProvider`.

## Решение
1. **Обновлен `_showLogoutDialog` в `more_screen.dart`:**
   - Добавлен `Consumer` для доступа к `ref`
   - Добавлен вызов `await ref.read(authProvider.notifier).logout()`
   - Убран ручной переход `context.go('/login')` - теперь роутер автоматически перенаправляет

2. **Добавлен импорт:**
   - `import '../../../auth/presentation/providers/auth_provider.dart';`

## Как это работает
1. Пользователь нажимает "Выйти"
2. Показывается диалог подтверждения
3. При подтверждении вызывается `authProvider.logout()`:
   - Удаляется токен из secure storage
   - Сбрасывается состояние на `AuthState()` с `isAuthenticated = false`
4. Роутер автоматически отслеживает изменение `isAuthenticated`
5. Срабатывает redirect логика в `app_router.dart` (строки 58-74)
6. Пользователь перенаправляется на `/login`

## Тестирование
1. Запустите приложение
2. Войдите в систему
3. Перейдите на вкладку "Еще"
4. Нажмите "Выйти"
5. Подтвердите выход
6. Вы должны автоматически попасть на экран входа
7. При попытке вернуться назад должна снова показаться страница входа
